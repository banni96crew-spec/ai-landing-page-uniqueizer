1. Основная проектная документация

    Основной PRD (этот документ): Выступает как единственный источник истины (Single Source of Truth) для генерации кода и ИИ-ассистента Cursor. При правке смысла или логики сначала обновляется этот файл, а затем зависимые документы.

    Инструкция по VPS-деплою: Пошаговое руководство для развертывания проекта на сервере (клонирование репозитория, настройка .env, запуск Docker).

2. Визуальные диаграммы и схемы (Экспорт)

Эти документы визуально дублируют разделы базовой логики и схемы данных:

    Архитектурные схемы: docs/architecture.md (для PR/диффов) и docs/html/architecture.html (для встраивания).

    Схемы базы данных: docs/database.md и docs/html/database.html.

3. Инфраструктура и развертывание (IaC)

Для обеспечения работы продукта в контейнеризованной среде требуются:

    Dockerfile: Описывает процесс сборки однослойного образа на базе Python 3.12-slim с установкой Node.js и системных зависимостей для Playwright.

    docker-compose.yml: Файл для оркестрации контейнеров, настройки портов (8000 для backend, 3000 для frontend) и монтирования томов (volumes) для SQLite и ZIP-архивов.

    entrypoint.sh: Bash-скрипт инициализации, который запускает одновременно FastAPI (uvicorn) и Next.js (npm run start) внутри одного контейнера.

    .env.example: Шаблон конфигурации переменных окружения (включая пути к БД, лимиты, таймауты, настройки CORS и базовые URL для API/WebSocket).

4. Управление зависимостями

    backend/requirements.txt: Спецификация зависимостей для Python-бэкенда (включая FastAPI, Playwright, BeautifulSoup4, OpenAI, Anthropic, Pillow).

    frontend/package.json: Конфигурационный файл для фронтенд-части, включающий зависимости для Next.js 14, React 18, TailwindCSS 4 и Framer Motion 11.

5. Скрипты базы данных

    migrations/001_init.sql: DDL-скрипт (Data Definition Language) для инициализации SQLite таблиц (jobs, artifacts, settings, logs), который обязательно должен копироваться в Docker-образ.
